Ship it with Ralph.

Turning a PRD into working code — one story, one commit, on repeat.

Chris Raethke @codesoda
What you'll leave with

Three takeaways.

  • Prep the loop. The agent is only as good as the context you hand it.
  • CLAUDE.md is the manual. One file decides whether your agent ships or drifts.
  • Trust the iterations. Small stories + a quality gate beats a megaprompt.
The Problem

Vibes don't deploy.

AI can write code. So why are our apps still prototypes?

The tension

Prototype to production is still a cliff.

  • Generated code looks fine — until it doesn't compile against your stack.
  • One megaprompt = one shot. If it misses, you rewrite from scratch.
  • No tests. No conventions. No memory of what you already decided.
Pick your tool

Different jobs, different agents.

  • Small bugs, tiny enhancements → Single-shot. "Change this one thing."
  • Tricky bugs, small-medium features → Plan mode. Plan first, then set it off.
  • New projects, full features (greenfield) → PRD + Ralph.← today
  • Brownfield, critical updates → Agent pairing (SEAMS). Human-in-the-loop.
Meet Ralph

An autonomous loop. One story at a time.

snarktank/ralph — picks the next unchecked story in your PRD, implements it, runs your tests, commits if they pass, repeats.

./scripts/ralph/ralph.sh --tool claude # default: 10 iterations
Part 1 — Prep

What Ralph reads.

Three files decide everything. Get these right, the rest is watching a progress bar.

File #1

CLAUDE.md — the operating manual.

  • Stack. Next.js, Supabase, Tailwind, pnpm. Non-negotiables.
  • Conventions. Server components by default. No any. No commented-out code.
  • Quality gate. pnpm typecheck && pnpm lint && pnpm test.
  • Scope discipline. Implement the current story. Nothing more.
File #2 (a folder, really)

agent-docs/ — the handbook.

agent-docs/
├── architecture.md   folder layout, rendering model, seams
├── database.md       supabase schema + access rules
├── language.md       TS / Next / React conventions
├── design.md         how to apply design/DESIGN.md
├── deployment.md     vercel + env vars
└── testing.md        vitest, playwright, CI gate
File #3

PRD.md → prd.json

Stories with passes: false. Ralph picks, implements, flips to true.

{
  "id": "US-002",
  "title": "Add-task form",
  "acceptanceCriteria": [
    "Form persists to supabase via server action",
    "Row appears in list",
    "Typecheck passes"
  ],
  "priority": 2,
  "passes": false
}
Part 2 — Run

The loop.

Ralph does the same five things, over and over, until the PRD is green.

Per iteration

Pick · build · test · commit · mark.

  • 1. Pick. Highest priority story where passes: false.
  • 2. Build. Implement that story. Nothing adjacent.
  • 3. Test. Typecheck + lint + unit tests. Any failure → retry.
  • 4. Commit. Conventional message, one story per commit.
  • 5. Mark. Flip passes: true. Next.
